# Phase 3: Advanced UI Interactions & Timeline

With Phase 1 handling data and Phase 2 placing the primary monitoring grid, Phase 3 aims at providing interactivity and completing the central 50" HUD components. We will add the Schedule Timeline (Gantt overlay), wire the Agent action controls to the backend API, and add a quick Command Palette (Cmd+K).

## Implementation Strategy

### Task 1: Schedule Timeline (Gantt Chart)
The central wide component showing upcoming outputs.
- [ ] Create `ScheduleTimeline.tsx` replacing the placeholder.
- [ ] Build a horizontal scrolling 24h timeline block plotting `mockTasks` across discrete time ticks.

### Task 2: Live Control Protocol (Action Wiring)
Enable the hover action buttons on Agent Cards.
- [ ] Update `AgentCard.tsx` buttons to dispatch `POST /api/agents/:id/command` actions.
- [ ] Add generic `useAgentCommand.ts` hook utilizing fetch to the API.

### Task 3: Performance Metrics Sparklines
Refining the Performance Metrics panel which currently has a basic placeholder.
- [ ] Create `PerformanceMetricsPanel.tsx` mapping success rates over 24h intervals.
- [ ] Inject into `DashboardShell.tsx`.

### Task 4: Command Palette (Cmd+K)
The HUD needs rapid keyboard-first navigation for human ops members.
- [ ] Implement `CommandPalette.tsx` overlay that captures `CMD+K`.
- [ ] Populate with quick actions: `Restart all Agents`, `Kill specific Agent`, `Toggle Theme`.
