# Phase 4: Decentralized Worker Client (Agent Node)

Phase 4 focuses on building the standalone Node.js client (`@omniscient/worker`) that will actually run on the physical mini-PCs. This worker is responsible for executing tasks, pinging the Control Plane with telemetry (memory/cpu), and listening to commands over Redis.

## Architecture Guidelines

- **Standalone Service**: Runs indefinitely via PM2 or straight Node.
- **Resiliency**: If the Control Plane is offline, the worker should continue doing its active task and queue up heartbeats or gracefully retry.
- **State Machine**: The worker is a finite state machine (`IDLE` -> `WORKING` -> `PAUSED`).
